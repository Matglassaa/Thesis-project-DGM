# Data Generation

This folder contains the scripts for generating the geological datasets using the Flumy package.

## File Structure

The main file for data generation is `dataset_generation_batch.py`. This is the only file that should be run to generate the datasets. The other python files are utility and worker scripts called by the main file.

- `dataset_generation_batch.py`: Main script for generating datasets in parallel batches.
- `dataset_generation_single.py`: A script for generating a single data sample. Useful for testing and debugging.
- `flumy_utils.py`: Contains utility classes for managing data and simulations.
- `flumy_worker.py`: Contains the core worker function for running a single Flumy simulation.
- `environment.yml`: Conda environment file for reproducing the required software environment.

## Python Scripts

### `dataset_generation_batch.py`

This is the main script to generate the dataset. It uses the `BatchSimulator` class from `flumy_utils.py` to run multiple Flumy simulations in parallel. The script is configured with parameters such as the total number of samples, batch size, and the number of parallel jobs.

### `dataset_generation_single.py`

This script is a simplified version of the batch generation script, designed to generate only a single sample. This is useful for testing the simulation with a specific set of parameters or for debugging the simulation process.

### `flumy_utils.py`

This module provides two main classes:

- **`FlumyDataManager`**: This class handles the creation and management of HDF5 files. It pre-allocates space for the data, which helps in efficient storage.
- **`BatchSimulator`**: This class orchestrates the parallel execution of the Flumy simulations. It uses `joblib` to manage the parallel processes.

### `flumy_worker.py`

This module contains the core logic for running a single simulation instance.

- **`run_simulation_batch`**: This function is the heart of the data generation process. It initializes a Flumy simulation with a unique seed, runs the simulation, processes the output facies, and saves the data to an HDF5 file.
- **`groupFacies`**: This utility function groups the detailed facies generated by Flumy into a smaller number of categories, which is more practical for training deep learning models.

## Output `.h5` File Structure

The generated data is stored in HDF5 (`.h5`) files. Each file represents a batch of simulations. The structure of each `.h5` file is as follows:

- **`facies`**: A 3D numpy array representing the facies model.
- **`grain`**: A 3D numpy array representing the grain size distribution.
- **`age`**: A 3D numpy array representing the geological age of the deposits.

Each dataset has a shape of `(n_samples, nx, ny, nz)`, where `n_samples` is the number of simulations in the batch, and `nx`, `ny`, `nz` are the dimensions of the grid.

## Environment Installation

To run the data generation scripts, you need to have a Conda environment with the required packages installed. You can create and activate the environment using the following commands:

```bash
conda env create -f environment.yml
conda activate Flumy_3_09
```
